4

Fill Out the Form Fields

Let's go through each field in detail:
Full Name REQUIRED
Type: Text

Purpose: Enter the candidate's complete legal name (first and last). The system will automatically create a Contact record using this name.

Example: John Michael Smith

Validation: Must not be empty.

Email Address REQUIRED
Type: Email

Purpose: The candidate's primary email address. This is used for communication and serves as the unique identifier to prevent duplicates.

Example: john.smith@email.com

Validation: Must be a valid email format. System automatically checks if this email already exists in your Contacts.

üö´ Duplicate Check: If this email already exists, you'll see a warning message preventing duplicate candidate creation. Update the existing record instead.
Phone Number OPTIONAL
Type: Phone

Purpose: The candidate's primary contact phone number. Becomes the main phone number on the Contact record.

Example: (555) 123-4567

Recommendation: Strongly recommended to fill this in for easy calling from Salesforce.

Position OPTIONAL
Type: Dropdown (Picklist)

Purpose: Specifies what role this candidate is being recruited for. Used for filtering and reporting.

Default: Insurance Agent

Available Options:
  • Insurance Agent (default) - Entry-level insurance sales position
  • Senior Insurance Agent - Experienced agent with track record
  • Insurance Sales Representative - Sales-focused role
  • Account Manager - Client relationship management
  • Team Lead - Leadership position managing agents
  • Sales Manager - Management role
  • Other - Any role not listed above
Lead Source OPTIONAL
Type: Dropdown (Picklist)

Purpose: Tracks where this candidate was found. Critical for measuring recruiting channel effectiveness.

Default: Recruiter Portal

Available Options:
  • Recruiter Portal (default) - Manually entered in system
  • LinkedIn - Imported from LinkedIn via browser extension
  • Indeed - Job board applicant
  • Monster - Job board applicant
  • Employee Referral - Referred by current employee
  • Company Website - Applied via careers page
  • Job Fair - Met at recruiting event
  • Other - Any source not listed above
üí° Best Practice: Always select the true source to help measure which recruiting channels generate the best candidates.
Initial Notes OPTIONAL
Type: Text Area (Long Text)

Purpose: Capture any important context, first impressions, or relevant background about the candidate.

Example: "Strong communication skills during initial phone screen. Has 5 years experience in financial services. Interested in insurance sales career transition."

Recommendation: Use this field to document anything you might forget later‚Äîfirst impressions matter!

Next Steps: Schedule an Interview

‚ö†Ô∏è Important: There is NO "Schedule a Call" checkbox in the Create Candidate modal. You must use the separate "Schedule Interview" button to book meetings.

‚ùå Common Mistakes to Avoid

Mistake #1: Not checking for duplicate emails before creating a candidate.
Solution: The system handles this automatically‚Äîif the email exists, you'll get a warning.
Mistake #2: Entering a nickname instead of full legal name.
Solution: Always use the candidate's complete legal name (first and last).
Mistake #3: Skipping Position and Lead Source fields.
Solution: Fill these in! They're critical for reporting and tracking channel effectiveness.
Mistake #4: Expecting to schedule a call from the same modal.
Solution: Candidate creation and interview scheduling are two separate steps with two separate buttons.
