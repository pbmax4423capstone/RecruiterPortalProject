Copilot Chat vs Agent Mode

Understanding the two modes

üí¨ Copilot Chat

Traditional conversational interface for asking questions:

  • Ask questions about your code
  • Get explanations of concepts
  • Generate code snippets to copy/paste
  • Debug specific issues
You: How do I sort an array of objects by date in JavaScript? Copilot: Here's how to sort by date: const sorted = items.sort((a, b) => new Date(a.date) - new Date(b.date) );

ü§ñ Copilot Agent Mode

Agent mode goes beyond chat‚Äîit can take action on your codebase:

  • Edit files directly in your workspace
  • Create new files and folders
  • Run terminal commands
  • Search your codebase to understand context
  • Make multi-file changes in one request
üí° When to Use Agent Mode Use Agent mode when you want Copilot to actually make changes to your code, not just suggest them.

Effective Prompting

Get better results from Copilot

üìù The CRISP Framework

Write better prompts with C-R-I-S-P:

Letter Meaning Example
Context What's the situation? "In our Salesforce LWC project..."
Role Who should Copilot be? "Act as a senior developer..."
Instructions What do you need? "Create a function that..."
Specifics Any requirements? "Use async/await, handle errors..."
Pattern What format/style? "Follow our existing code style..."

‚ùå Bad vs ‚úÖ Good Prompts

Example 1: Too Vague

‚ùå Bad: "Fix the bug" ‚úÖ Good: "The login form submits twice when users click the button quickly. Add debouncing to prevent duplicate submissions. The form is in src/components/LoginForm.js"

Example 2: Missing Context

‚ùå Bad: "Write a test" ‚úÖ Good: "Write a Jest unit test for the calculateTotal function in src/utils/pricing.js. Test cases should include: - Normal calculation with valid inputs - Zero quantity edge case - Negative price validation"

Example 3: Unclear Output

‚ùå Bad: "Make it better" ‚úÖ Good: "Refactor getUsers() to use async/await instead of .then() callbacks. Keep the same functionality but improve readability. Add error handling with try/catch."
‚úÖ Pro Tip The more specific you are, the better results you'll get. Include file names, function names, and specific requirements.

Agent Mode in Action

Real-world use cases

üöÄ Common Agent Mode Tasks

1. Create New Features

You: Create a new user profile page component that displays: - User avatar - Name and email - Join date - Recent activity list Use our existing Card component and follow the styling in ProfilePage.css. Create both the component and a test file.

2. Refactor Existing Code

You: Refactor the UserService class to use dependency injection instead of hardcoded database connections. Update all files that import UserService to use the new pattern.

3. Fix Bugs Across Multiple Files

You: The date picker is showing dates in the wrong timezone. Find all places where we format dates and ensure they use the user's local timezone. Check components/, utils/, and api/.

‚ö° Agent Mode Workflow

  1. Describe your task - Be specific about what you need
  2. Review the plan - Agent will show you what it intends to do
  3. Approve or adjust - You can modify before it executes
  4. Watch it work - Agent makes changes in real-time
  5. Review changes - Always review before committing
‚ö†Ô∏è Always Review AI-Generated Code Copilot is helpful but not perfect. Always review changes before committing. Check for:
  • Logic errors
  • Security issues
  • Code that doesn't match your patterns
  • Missing edge cases

Best Practices

Using Copilot responsibly

‚úÖ Do's

  • Review all generated code - Never blindly accept suggestions
  • Test thoroughly - AI can write buggy code too
  • Use for boilerplate - Great for repetitive code patterns
  • Iterate on prompts - Refine if first result isn't right
  • Learn from suggestions - Discover new patterns and APIs
  • Break down complex tasks - Smaller requests = better results

‚ùå Don'ts

  • Don't share secrets - Never put API keys or passwords in prompts
  • Don't blindly trust - Verify security-sensitive code
  • Don't skip understanding - Know what the code does before using it
  • Don't use for everything - Simple code is often faster to write yourself
  • Don't ignore context - Copilot doesn't always understand your full codebase

üéØ Team Collaboration with Copilot

When multiple developers use Copilot:

  • Share effective prompts - Build a team knowledge base
  • Standardize patterns - Agree on code styles Copilot should follow
  • Review AI code in PRs - Extra scrutiny on generated code
  • Document AI contributions - Note when significant code was AI-generated
üí° Copilot Limitations
  • May not know your latest code changes
  • Can hallucinate APIs that don't exist
  • May suggest outdated patterns
  • Doesn't run or test the code
