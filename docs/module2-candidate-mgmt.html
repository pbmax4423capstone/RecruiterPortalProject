‚ûï Methods to Create Candidates

Required Fields

Understanding which fields are mandatory and why

üìã Candidate Field Reference

The following table shows all candidate fields and their requirements:

Field Name Required Description
First Name Yes Candidate's first name
Last Name Yes Candidate's last name
Email Yes Primary email address

  • Highest Level - Updated automatically as interviews complete

Pipeline Stages

Understanding the candidate pipeline and stage progression

üìä Pipeline Stage Overview

Candidates move through defined stages as they progress through the recruiting process:

New ‚Üí In Process ‚Üí Hired

or ‚Üí Inactive (at any point)

üì∏
Screenshot: Pipeline Stage Selection

üè∑Ô∏è Stage Definitions

Stage When to Use Typical Duration
New Candidate just added, not yet contacted 1-3 days
In Process Active engagement - interviews scheduled or in progress 2-4 weeks
Hired Candidate accepted offer and hired Final stage
Inactive Candidate withdrawn, rejected, or no longer pursuing Final stage

üìà Highest Level Achieved

In addition to pipeline stage, track the furthest interview level reached:

  • None - No interviews completed
  • Initial Screen - Completed first phone/video screen
  • Manager Interview - Met with hiring manager
  • Final Interview - Completed final round
  • Offer Extended - Received formal offer
üí° Best Practice Update the pipeline stage immediately when status changes. This ensures accurate dashboard metrics and reporting.

Source Tracking

Understanding how to track where candidates come from

üîç Why Source Tracking Matters

Tracking how candidates are sourced helps the organization understand which recruiting channels are most effective. This data drives budget decisions and recruiting strategy.

Available Source Options

Source When to Use
LinkedIn Found via LinkedIn search or imported via extension
Referral Referred by current employee or agent
Job Board Applied through Indeed, ZipRecruiter, etc.
Career Fair Met at recruiting event
Website Applied through company career site
ZoomInfo Sourced from ZoomInfo database
Other Any other source (add details in notes)

üìä Source Metrics

The system tracks these metrics by source:

  • Volume - Number of candidates from each source
  • Conversion Rate - Percentage that become hires
  • Time-to-Hire - Average days to hire by source
  • Quality Score - Performance rating of hires by source
üí° Referral Tracking When selecting "Referral" as the source, you can specify who made the referral in the Referred By field for bonus tracking.

Best Practices

Tips for effective candidate management

‚úÖ Do's

  • Update records promptly - Keep status current after each interaction
  • Use accurate sources - Select the correct source for reporting accuracy
  • Add detailed notes - Document important conversations and observations
  • Verify contact info - Confirm email and phone before saving
  • Schedule follow-ups - Create tasks for pending actions

‚ùå Don'ts

  • Don't create duplicates - Search before creating new candidates
  • Don't leave status stale - Inactive candidates should be marked Inactive
  • Don't skip required fields - Incomplete records cause reporting issues
  • Don't use "Other" source excessively - Be specific when possible
  • Don't delete records - Mark as Inactive instead for audit trail

üéØ Quality Checklist

Before considering a candidate record complete, verify:

  • ‚òê Name spelled correctly
  • ‚òê Valid email address
  • ‚òê Phone number with area code
  • ‚òê Correct source selected
  • ‚òê Status reflects current stage
  • ‚òê Highest level updated after interviews
‚ÑπÔ∏è Data Quality Reports Managers run weekly reports to identify incomplete records. Keep your records updated to avoid follow-up requests.
