Candidate Management

TC-CAND-001 Create New Candidate with Required Fields High

User logged into Salesforce with Recruiter profile. On Candidates tab or Dashboard.

  1. Click "New Candidate" button
  2. Enter First Name: "Test"
  3. Enter Last Name: "Candidate"
  4. Enter Email: "test.candidate@example.com"
  5. Enter Phone: "(555) 123-4567"
  6. Select Source: "Referral"
  7. Click "Save"
Expected Result: Candidate record created successfully. User redirected to candidate detail page. Status defaults to "New". All entered fields display correctly.
TC-CAND-002 Required Field Validation High

User on New Candidate form.

  1. Leave First Name blank
  2. Leave Last Name blank
  3. Click "Save"
Expected Result: Save fails. Error messages displayed for required fields (First Name, Last Name). Form remains open for correction.
TC-CAND-003 Update Candidate Information Medium

Existing candidate record open in edit mode.

  1. Click "Edit" button on candidate record
  2. Change Phone to "(555) 987-6543"
  3. Add Notes: "Updated contact information"
  4. Click "Save"
Expected Result: Record saved successfully. Updated fields display new values. Last Modified date/user updated.
TC-CAND-004 Change Candidate Pipeline Stage High

Candidate exists with Status = "New".

  1. Open candidate record
  2. Click on Status field or path component
  3. Change status from "New" to "Phone Screen"
  4. Save the record
  5. Repeat for each stage: Phone Screen ‚Üí Interview Scheduled ‚Üí Interview Complete ‚Üí Offer ‚Üí Hired
Expected Result: Status updates correctly at each stage. Visual indicators (badges/colors) update appropriately. Pipeline stage change is reflected on Dashboard.
TC-CAND-005 Duplicate Detection on Email Medium

Candidate exists with email "existing@example.com".

  1. Click "New Candidate"
  2. Enter different name but same email: "existing@example.com"
  3. Fill other required fields
  4. Click "Save"
Expected Result: System detects duplicate. Warning/alert shown with link to existing record. User can choose to proceed or cancel.
TC-CAND-006 Source Field Tracking Medium

User on New Candidate form.

  1. Create candidate with Source = "Referral", save
  2. Create candidate with Source = "ZoomInfo", save
  3. Create candidate with Source = "LinkedIn", save
  4. Create candidate with Source = "Job Board", save
  5. View Dashboard or run Source report
Expected Result: Each source value saved correctly. Source breakdown on Dashboard accurately reflects counts. Reports can filter by source.
TC-CAND-007 Delete Candidate Record Low

Test candidate record exists with no related interviews.

  1. Open candidate detail page
  2. Click dropdown arrow ‚Üí "Delete"
  3. Confirm deletion in dialog
Expected Result: Record deleted successfully. User redirected to list view. Deleted record no longer appears in searches or dashboard counts.
TC-CAND-008 Search and Filter Candidates Medium

Multiple candidates exist with various statuses and sources.

  1. Go to Candidates list view
  2. Use global search to find candidate by name
  3. Filter list view by Status = "Interview Scheduled"
  4. Sort by Created Date descending

Interview Tracking

TC-INT-001 Schedule Phone Interview High

Candidate exists with Status = "New" or "Phone Screen".

  1. Open candidate record
  2. Click "Schedule Interview" or "New Interview" button
  3. Select Interview Type: "Phone Screen"
  4. Enter Date/Time for tomorrow at 10:00 AM
  5. Add Notes: "Initial phone screen"
  6. Click "Save"
Expected Result: Interview record created. Linked to Candidate. Interview appears in related list. Candidate status updates to "Interview Scheduled" if configured.
TC-INT-002 Schedule In-Person Interview High

Candidate has completed phone screen successfully.

  1. Open candidate record
  2. Click "New Interview"
  3. Select Interview Type: "In-Person"
  4. Enter Date/Time for next week
  5. Enter Location: "123 Main St, Conference Room A"
  6. Assign Interviewer (if field exists)
  7. Click "Save"
Expected Result: In-person interview record created with all details. Location field saved. Multiple interview records can exist for same candidate.
TC-INT-003 Record Interview Completion - Pass High

Interview record exists with Status = "Scheduled". Interview date has passed.

  1. Open the scheduled interview record
  2. Click "Edit" or "Log Interview"
  3. Change Status to "Completed"
  4. Set Result/Outcome: "Pass" or "Proceed"
  5. Enter Feedback: "Strong candidate, good communication skills"
  6. Click "Save"
Expected Result: Interview marked complete. Completion date auto-populated. Feedback saved. Candidate status may auto-update based on workflow.
TC-INT-004 Record Interview Completion - No Hire Medium

Interview exists. Candidate did not pass the interview.

  1. Open the interview record
  2. Change Status to "Completed"
  3. Set Result/Outcome: "No Hire" or "Rejected"
  4. Enter Feedback explaining decision
  5. Save the record
  6. Check if candidate status auto-updates
Expected Result: Interview completed with rejection outcome. Candidate status updates to "Rejected" or remains for manual update. Feedback preserved for compliance.
TC-INT-005 Reschedule Interview Medium

Interview exists with future date/time. Candidate needs to reschedule.

  1. Open the interview record
  2. Click "Edit"
  3. Change Date/Time to new date
  4. Update Notes: "Rescheduled per candidate request"
  5. Save the record
Expected Result: Interview date/time updated. Notes reflect change. Any calendar integrations update accordingly.
TC-INT-006 Cancel Interview Low

Scheduled interview exists. Need to cancel (candidate withdrew, etc.).

  1. Open interview record
  2. Change Status to "Cancelled"
  3. Enter cancellation reason in Notes
  4. Save the record
Expected Result: Interview status changes to Cancelled. Record preserved for history. Does not count in completed interview metrics.
TC-INT-007 Interview Metrics on Dashboard High

Multiple interviews exist: some scheduled, some completed this week/month.

  1. Navigate to Recruiter Dashboard
  2. Locate Interview metrics section
  3. Verify "This Week" count matches actual completed interviews
  4. Verify "This Month" count matches actual completed interviews
  5. Verify "Scheduled" count matches upcoming interviews
Expected Result: All interview metrics accurate. Counts update in real-time or on refresh. Clicking metrics navigates to filtered list.

Contract B Lifecycle

‚ö†Ô∏è Critical Contract B Metrics

Verify all tests use correct values: 4 months duration, $2,500 FYC requirement, 5 submissions required, 4 month max extension.

TC-CTB-001 Contract B Record Creation High

Candidate hired and ready for Contract B onboarding.

  1. Create new Contract B record for advisor
  2. Set Start Date to today's date
  3. Verify End Date auto-calculates to 4 months from start
  4. Verify FYC Requirement shows $2,500
  5. Verify Submission Requirement shows 5
  6. Save the record
Expected Result: Contract B record created. End Date = Start Date + 4 months. FYC Requirement = $2,500. Submission Requirement = 5. Status defaults to "Active".
TC-CTB-002 FYC Tracking and Rollup High

Active Contract B record exists with $0 FYC.

  1. Add/update production record with FYC = $1,000
  2. Verify Contract B FYC total updates to $1,000
  3. Add another production record with FYC = $1,500
  4. Verify Contract B FYC total = $2,500
  5. Verify progress indicator shows 100% of $2,500 requirement
Expected Result: FYC correctly rolls up from production records. Total matches sum of individual FYC values. Progress shows accurate percentage toward $2,500 goal.
TC-CTB-003 Submission Count Tracking High

Active Contract B with 0 submissions.

  1. Add a submission record linked to Contract B
  2. Verify submission count = 1
  3. Add 4 more submissions
  4. Verify submission count = 5
  5. Verify progress shows 100% of 5 submission requirement
Expected Result: Submission count accurately reflects linked records. Progress indicator shows 5/5 when requirement met.
TC-CTB-004 Status Indicator - On Track High

Contract B in month 2 of 4. FYC = $1,500 (60%). Submissions = 3 (60%).

  1. Open Contract B record
  2. Verify status indicator displays "On Track" (green)
  3. Verify dashboard shows this advisor in "On Track" category
Expected Result: With 50% time elapsed and 60% progress, status shows "On Track" with green indicator.
TC-CTB-005 Status Indicator - At Risk High

Contract B in month 3 of 4. FYC = $1,000 (40%). Submissions = 2 (40%).

  1. Open Contract B record
  2. Verify status indicator displays "At Risk" (yellow/orange)
  3. Verify dashboard shows this advisor in "At Risk" category
Expected Result: With 75% time elapsed but only 40% progress, status shows "At Risk" with yellow/orange indicator.
TC-CTB-006 Status Indicator - Critical High

Contract B in final month (month 4). FYC = $500 (20%). Submissions = 1 (20%).

  1. Open Contract B record
  2. Verify status indicator displays "Critical" (red)
  3. Verify dashboard shows this advisor in "Critical" category
  4. Verify any alerts/notifications trigger appropriately
Expected Result: With contract ending soon and requirements far from met, status shows "Critical" with red indicator.
TC-CTB-007 Grant Extension (Max 4 Months) High

Contract B nearing end date. Has not met requirements. No prior extensions.

  1. Open Contract B record
  2. Click "Grant Extension" or edit End Date
  3. Enter 2 month extension
  4. Verify new End Date = original + 2 months
  5. Try to add another 3 month extension
  6. Verify system limits total extension to 4 months max
Expected Result: Extension granted. End date updated. System enforces 4 month maximum extension limit. Cannot exceed 8 months total (4 original + 4 extension).
TC-CTB-008 Contract B Outcomes - Transition/Extension/Termination High

Three Contract B records: one met requirements, one extended, one failed.

  1. Test Transition: Contract B with $2,500+ FYC and 5+ submissions ‚Üí Mark as "Transition to A"
  2. Verify advisor record updates appropriately
  3. Test Extension: Contract B not meeting requirements ‚Üí Grant extension
  4. Verify status reflects extension period
  5. Test Termination: Contract B expired without meeting requirements ‚Üí Mark as "Terminated"
  6. Verify termination date and reason recorded
Expected Result: All three outcomes process correctly. Transition updates advisor status. Extension updates dates. Termination records final status with date.

  4. Verify "This Month's Hires" card shows accurate count
  5. Verify "Scheduled Interviews" card shows upcoming interviews
Expected Result: All summary cards display accurate, real-time data. Numbers match actual record counts in Salesforce.
TC-DASH-002 Pipeline Stage Breakdown High

Candidates exist in various pipeline stages.

  1. View pipeline breakdown section on Dashboard
  2. Verify counts for each stage: New, Phone Screen, Interview Scheduled, Interview Complete, Offer, Hired
  3. Click on a stage to view filtered list
  4. Confirm filtered list matches the count shown
Expected Result: Each pipeline stage shows accurate count. Clicking navigates to correctly filtered list view.
TC-DASH-003 YTD Metrics Accuracy Medium

Records exist from current calendar year.

  1. View YTD (Year-to-Date) metrics section
  2. Verify YTD Hires count matches records hired since Jan 1
  3. Verify YTD Interviews matches completed interviews since Jan 1
  4. Run report to cross-check numbers
Expected Result: YTD metrics accurately reflect all activity from January 1 of current year through today.
TC-DASH-004 Pipeline Table Sorting Medium

Pipeline table visible with multiple candidates.

  1. Click column header "Name" to sort alphabetically
  2. Click again to reverse sort order
  3. Click "Status" column to sort by pipeline stage
  4. Click "Created Date" to sort by date
Expected Result: Table sorts correctly by each column. Sort indicator shows direction. Data re-orders appropriately.
TC-DASH-005 Action Buttons Functionality Medium

Dashboard loaded with candidate records visible.

  3. Cancel and return to Dashboard

  5. Verify interview scheduling form opens with candidate pre-selected

Candidates exist in all pipeline stages.

TC-DASH-007 Dashboard Refresh/Real-time Updates Medium

Dashboard open. Second browser/tab available for making changes.

  1. Note current counts on Dashboard

LinkedIn Extension

‚ÑπÔ∏è Prerequisites

LinkedIn extension installed in Edge or Chrome. User logged into Salesforce in same browser. LinkedIn account accessible.

TC-LI-001 Extension Installation Verification High

Extension files available. Browser supports extensions.

  1. Open browser extensions page (edge://extensions or chrome://extensions)
  2. Enable Developer Mode
  3. Load unpacked extension from provided folder
  4. Verify extension icon appears in toolbar
  5. Pin extension to toolbar if needed
Expected Result: Extension loads without errors. Icon visible in browser toolbar. No console errors on installation.
TC-LI-002 Salesforce Connection High

Extension installed. Salesforce credentials available.

  1. Click extension icon
  2. Click Settings/gear icon
  3. Enter Salesforce org URL
  4. Click Connect
  5. Log in with Salesforce credentials when prompted
  6. Grant permissions if requested
Expected Result: Connection established. "Connected" status displayed. Extension ready to import candidates.
TC-LI-003 Import Candidate from LinkedIn Profile High

Extension connected to Salesforce. On a LinkedIn profile page.

  1. Navigate to a LinkedIn profile page
  2. Click the extension icon
  3. Verify name and headline are captured
  4. Verify location is parsed
  5. Click "Import to Salesforce"
  6. Verify success message appears
  7. Click link to open new candidate record
Expected Result: Candidate created in Salesforce with: First/Last Name, Title/Headline, Location, LinkedIn URL, Source = "LinkedIn", Status = "New".
TC-LI-004 Duplicate Detection Medium

Candidate already exists from previous import.

  1. Navigate to same LinkedIn profile previously imported
  2. Click extension icon
  3. Attempt to import again
Expected Result: System detects duplicate. Warning message displayed with option to view existing record or proceed with new record.
TC-LI-005 Error Handling - Not on Profile Page Low

Extension connected. Not on a LinkedIn profile page.

  1. Navigate to LinkedIn search results page (not a profile)
  2. Click extension icon
  3. Observe behavior
Expected Result: Helpful error message displayed: "Please navigate to a LinkedIn profile page to import a candidate." No crash or confusing errors.
